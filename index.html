Temporal Difference General Game Player

Rather than writing an program that can play a single game well, why not write an AI that can play any game well?

My program is able to read games in XML format (The XML must follow the included game.xsd grammar). You can run the game by running start.py It will then allow you to select the players (human, random, or AI), at which point, it will run the game. The AI needs training to learn how to play well. To train the AI, run learningplayer.py.

A sample game, Dominion, has been included, and is described in dominion.py.

Capabilities:

  • Capable of playing non-deterministic games with hidden state. Doesn't cheat to win.
  • Able to learn different types of choices (For example, Ticket to ride has "Select turn action", "Select ticket", and "Draw card" choices).
  • Capable of handling choices with different options available each time, and choices that allow selecting multiple options.
  • Relatively fast runtime. Random players can finish a game in about 5 seconds, AI players can finish a game in about 20 (On my computer)
  • After 3,000 training games, wins against a random player about 80% of the time in Dominion (a quite complex game)

Limitations

  • Doesn't learn deterministic games as fast, because the AI usually selects the same moves for all players while training.
  • Can't handle choices that are not enumerable (For example, Monopoly bidding on properties)
  • Plays position-dependent games poorly (For example, in chess, two positions can be very similar, but have drastic differences in actual power)
  • Plays hidden-state-depending games poorly (For example, the AI doesn't try to guess your hand in Poker)
  • AI currently doesn't take into account recent moves players have taken (In risk, if my opponent just moved armies closer to me, he is likely attacking)
